# API contracts

Runtime OpenAPI: `/openapi/v1.json`.

Implemented resources include current user, accounts, dashboard messages/knowledge, orders CRUD/duplicate/validate/submit/export/PDF, product search/suggestions/details/favorites, and AI assistant messages. Standard failed submissions use Problem Details and every response includes `X-Correlation-ID`. Submission accepts `Idempotency-Key`; repeat calls to an already submitted order return the original confirmation identifiers.

Structured assistant endpoint: `POST /api/agent/messages`.
Assistant health endpoint: `GET /api/agent/health`.

- Input: `message`, optional `conversationId`, optional `orderId`, optional `history`.
- Output: structured `status`, `intent`, `confidence`, `entities`, `missingFields`, `clarificationQuestions`, `toolCalls`, `grounding`, `products`, `policy`, `version`, and `trace` metadata.
- Assistant analysis is generated by configured AI API settings (`Agent:Endpoint`, `Agent:Deployment`, `Agent:ApiKey`) and schema-constrained JSON output.
- Secrets are resolved from .NET configuration providers and environment variables; API keys are not stored in `appsettings*.json`.
- Recommended local secret setup:
	- `dotnet user-secrets set "Agent:Endpoint" "https://<resource>.openai.azure.com" --project CPP/apps/api/Cpp.Api.csproj`
	- `dotnet user-secrets set "Agent:Deployment" "<deployment-name>" --project CPP/apps/api/Cpp.Api.csproj`
	- `dotnet user-secrets set "Agent:ApiKey" "<key>" --project CPP/apps/api/Cpp.Api.csproj`
- Supported environment fallbacks include `AZURE_OPENAI_ENDPOINT`, `AZURE_OPENAI_DEPLOYMENT` (or `AZURE_OPENAI_CHAT_DEPLOYMENT`), and `AZURE_OPENAI_API_KEY`.
- `GET /api/agent/health` performs a live AI probe and validates schema-parse compatibility for startup/readiness checks.
- Approved tool calls are limited to read-only lookup operations and never execute transactional changes.
- Submit/confirm requests are intentionally handled as unsupported in assistant mode so transaction control stays in application workflows.
- Prompt-injection patterns are blocked and surfaced through policy/tracing fields.

Legacy route `POST /api/assistant/messages` has been removed.

Product search query parameters: `criterion`, `q`, and `favorites`. Criteria are Product Name, Item Number, Active Ingredient, Product Category, GTIN, Package Size, Vendor/Supplier, and Product Line.

Order writes contain account IDs, customer PO, contact/shipping fields, freight, requested arrival date, and exact SKU lines. Validation returns `severity`, `code`, `message`, `field`, and `suggestedResolution`.
